Rsa 非对称加密 优点：1.避免私钥在传递给加密者时丢失。2.n足够大时分解时间过长保证了rsa加密传递信息的安全性
RSA加密流程：不妨假设想要收集信息的端为A，想要传递信息的一端为B，在想要收集信息时，A首先要生成两把钥匙，分别叫做公钥和私钥，在RSA加密解密的过程中，公钥用于加密内容，私钥用于解密内容，而由于只需要传递给B公钥，从而避免了私钥的丢失，也就是避免了对称加密传递密钥时产生的意外导致密钥泄漏，从而避免了加密信息的泄露（这是第一个非对称加密相对于对称加密的优点）。而且一对公钥与一对私钥是相匹配的，也就是仅有A手中的私钥才能解开由A所传出去的公钥所加密的内容。
RSA的加密解密过程中还需注意的是，由于RSA的加密基于当前计算机算力的有限，本质上是一种通过数学运算产生的加密解密方式，所以从明文到密文的加密和从密文到明文的解密过程中，都需要提前或者在最后将字符串或者数字进行对应的转换，举例说明就是A若想要加密一段字符“我是谁”，首先要将“我是谁”这段字符转化为整型数字编码的形式，通过RSA加密形成密文（一个整数），再通过RSA解密将密文变文加密前的明文（即上文提到的那串“整型数字”），最后将这段数字再通过一定方式（这种方式对于A与B均为公开的，即使是第三者知道也没什么关系，因为RSA加密的保密在于大质数的分解，而不是最终对字符串的编码。但是不排除有通过字符串长短攻击的第三者。）对整形数字解码，最终得到B想要传递给A的密文。
已知公钥和私钥都由A产生，那么A需要完成的事情有如下几点：
1，公钥以及私钥中n的生成：随机挑选大的质数p与q，且两者不相等，同时计算两者的乘积，令 n = pq
2，再进行第二步运算phi（n）= (p-1)(q-1)
3，生成公钥中的e：e需要同时满足1，1<e<phi(n). 2, gcd(e,phi(n))= 1，即e与phi（n）互质。
4，生成私钥d，需要满足的公式为 ed = kphi(n)+1 (此处k为整数，即d为私钥)
5，传递公钥，保存私钥，消除pq（不泄漏就行）。
#################################
###一个简单的rsa加密解密实现过程
import gmpy2,libnum,math
from Crypto.Util.number import *
from Crypto import *
message_test = b’Lrene’ # 设定所传递的信息，由B产生
m = bytes_to_long(message_test) # 将信息转换为整数
p = getPrime(512) # 获取第一个512位长的质数p
q = getPrime(512) # 获取第二个512位长的质数q
n = pq # 计算n的取值
e = 65537 # 生成公钥中的e，需要满足:1<e<φ（n）&gcd(e,phi) = 1
phi = (p-1)(q-1) # 运用欧拉函数生成φ（n）
c = pow(m,e,n) #!此处为加密过程：运用数学运算 c = m^e mod(n)
d = gmpy2.invert(e, phi) # 生成私钥中的 d，由A进行保存，此处涉及到的是e*d=phi（n）+1
print(‘c=’,c)
print(‘m=’,m)
print(‘d=’,d)
‘’’
以上为加密过程
‘’’

解密过程 此处的解密为A获取到密文后的过程

decode_message = pow(c,d,n) # 此处为解密过程，运算为：decode_message = c^d mod n
print(‘decode_message=’,decode_message)
print(long_to_bytes(pow(c, d, n)))
‘’’
将这段代码运行可以发现对于配置较低的电脑运行还是有一点点卡顿，当加大p、q的长度会更卡
根据RSA的加密过程可以发现，想要解密信息需要知道c,d,n然后运行数学公式：m = c^d mod n，而
n即使被第三者知道，也需要获取d（私钥）才能继续解密。观察d的过程可以发现 ed = kphi（n）+1
此时需要去解phi（n）,而phi（n）的形成需要p、q，最终涉及到n的分解。
RSA安全性在我看来是由于算力的限制，如果n的位数大到一等程度就会使分解n的过程耗时极长
从而保证安全性.
从另一方面说如果有人能拥有即刻分解无限位数的质因数分解能力，第三方即可通过分解n从而得到p、q、phi(n),d,
进一步通过e解出d，从而获取完整私钥。
